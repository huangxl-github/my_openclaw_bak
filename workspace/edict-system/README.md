# 🏛️ 三省六部制 AI 多 Agent 协作系统

> 用唐朝 1300 年前的帝国制度重新设计 AI 多 Agent 架构 —— 分权制衡、可观测、可审计、可干预。

          │
     ┌───▼────┐
     │ 👑 太子 │ 消息分拣、需求整理、标题概括

## 🎯 核心特性

| 功能模块 | 说明 |
|---------|------|
| 🚫 **门下省审核** | 制度性封驳机制 —— 不是做完就交，必须达标才放行 |
| 👁️ **完全可观测** | 每个 Agent 独立 Workspace，完整流转链可追溯 |
| ⚡ **实时干预** | 支持叫停/取消/恢复任务 |
| 📋 **奏折存档** | 所有任务记录归档为 Markdown 文档 |
| 🔌 **飞书集成** | 天然支持飞书作为消息入口 |

## 🚀 快速开始

### 1️⃣ 安装要求

- ✅ OpenClaw（您已安装）
- ✅ Python 3.9+
- ⚠️ Node.js 18+（可选，用于前端看板）

### 2️⃣ 激活 Agent

```bash
# 每个 Workspace 都包含独立配置
openclaw agent list    # 查看所有已注册 Agent
openclaw agent start --agent taizi  # 手动启动某个 Agent
```

### 3️⃣ 下旨测试

通过飞书或其他消息渠道发送：
```
旨意：帮我分析这个项目并生成技术文档
```

流程：太子分拣 → 中书规划 → 门下审核 → 尚书派发六部 → 回奏
