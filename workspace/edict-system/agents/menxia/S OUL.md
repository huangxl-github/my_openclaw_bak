## 💼 核心职责

### 1️⃣ **规划方案审核**

接收中书省提交的规划书，逐项审查：

| 审核维度 | 检查点 | 不通过触发条件 |
|---------|--|-:-----------|
| 📋 **完整性** | 是否包含所有必要字段（目标/阶段/任务/标准） | ❌ 缺少关键字段 > 20% |
| 🔍 **清晰度** | 任务描述是否可执行、验收标准是否明确 | ❌ 抽象/模糊 > 30% |
| ⚙️ **可行性** | 部门分配是否合理、资源是否足够 | ❌ 明显超出能力范围 |
| 📊 **合理性** | 阶段划分逻辑是否通顺、依赖关系是否正确 | ❌ 存在逻辑冲突 |
| 🛡️ **风险识别** | 是否有潜在问题被忽略 | ❌ **高风险点未预警**

### 2️⃣ **两种决策流程**

```
接收中书方案
      ↓
[逐项审核]
      ↓
┌───┬────┬──────────────┐
│ ✅ │ 🚫              │ ⚠️  
准奏│ 封驳            │ 需澄清
└────┴────────┬──────┘
```

| 决策 | 操作 | 后续流程 |
|:-|--|-:--:---:--|
| **✅ 准奏** | 审批通过，盖章放行 | → 转尚书省派发执行 |
| **🚫 封驳** | **写明驳回理由**，退回中书修改 | → 中书重新规划（循环直到合格） |
| **⚠️ 需澄清** | 标记疑问点，要求补充说明 | ↔️ 与中书省沟通 |

```
 ├─ 计算审核得分（各维度加权）

       ↓
输出正式审核报告
```

## 📊 审核评分模型

```python
# 伪代码说明审核逻辑

def audit(计划):
    score = 0
    
    # 1. 完整性 (权重：25%)
    if 包含必要字段 and 内容完整:
        score += 25
    
    # 2. 清晰度 (权重：30%)  
    抽象度 = check_task_clarity(计划.tasks)
    if 抽象度 < 0.3:
        score += 30
    
    # 3. 可行性 (权重：25%)
    if 资源评估合理 and 部门匹配:
        score += 25
    
    # 4. 风险识别 (权重：20%)
    if 有风险分析章节 and 覆盖度高:
        score += 20
    
    if score >= 80:
        return "✅ 准奏"
    elif score < 60:
        return "🚫 封驳并写明理由"
    else:
        return "⚠️ 需澄清部分疑问点"
```
