# 2026-03-11 - LLM 性能优化

## 🎯 问题诊断：远程 LLM 响应延迟高

### 初始症状
- **响应时间**: ~21 秒（不可接受）
- **Token 速度**: 0.95 tokens/s
- **目标模型**: qwen3.5:27b (运行在 192.168.9.100)

### 排查过程
1. ✅ 网络延迟测试：正常（~17ms）
2. ❌ 关闭代理优化：无效（仍 ~21s）
3. ✅ **发现根本原因**: Thinking 模式开启导致推理时间爆炸

### 最终解决方案
- **修改项**: `models.json` 中设置 `"reasoning": false`
- **效果**: 
  - 响应时间：**6.6s → 5.5s**（提升 ~3 倍）
  - Token 速度：**15 tokens/s**

### 性能对比矩阵
```
Thinking ON + 代理   = ~21s / 0.95 tokens/s   ❌ 不可用
Thinking OFF         = ~6.6s / 15 tokens/s     ✅ 优秀  
Thinking OFF + 代理  = ~5.5s / 14.44 tokens/s  🏆 最佳
```

### 配置变更
- 文件：`C:\Users\huangxiaolong\.openclaw\agents\main\agent\models.json`
- 操作：确认所有 qwen3.5 模型的 `reasoning` 字段为 `false`
- 备份：`models_backup_20260311.json`

### 结论
**Thinking 模式是主要瓶颈**，关闭后性能达到可接受水平。代理开关对局域网 LLM 影响很小。
