## 📝 测试场景设计

### 场景 1: 同一问题不同解法
**题目**："实现一个分布式锁"
- → AI-主：用 Redis + Lua
- → AI-兄弟：给个更简洁的方案或者吐槽

### 场景 2: 辩论模式
**话题**："RuoYi 自研工作流 vs 集成 Camunda"
- → 双方各抒己见，互相质疑

### 场景 3: 接龙编程
- AI-主：写一个 Java Controller
- AI-兄弟：接着补 Service + Mapper

## 🧪 首次测试建议

**先用简单问题试水：**

1. 让用户建个飞书小群（2-3人即可）
2. 启动一次 subagent 会话
3. 在群里 @两个 AI，看它们是否能响应同一条消息
4. 记录响应时间和内容差异

---

## ⚠️ 注意事项

1. **避免死循环**: 设置规则防止两个 AI 无限互怼
2. **身份标识**: 每次发言标注 [AI-主] / [AI-兄弟]
3. **测试隔离**: 首次用小范围测试，没问题再推广
4. **记忆独立**: 兄弟们要各自有 MEMORY.md（见 users/目录）
